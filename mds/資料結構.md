# 資料結構

> **這份是什麼**：Firestore 上各集合/欄位的 schema 與相容性 fallback。改 schema 前先看這份、改完更新這份。
> **寫什麼**：欄位名、型別、預設值、可能是 undefined 的欄位、為什麼這樣設計
> **不寫什麼**：UI 邏輯、業務規則（去 [`功能說明.md`](功能說明.md)）

## driveData.studentList（學生主清單）

每位學生一個物件，欄位如下（2026-06-01 起）：

| 欄位 | 型別 | 預設 | 說明 |
|---|---|---|---|
| `id` | number | 必填 | `Date.now()` 產生，唯一識別 |
| `name` | string | 必填 | 學生姓名 |
| `grade` | string | 必填 | 國小/國一/國二/國三/高一/高二/高三/大學（見 `GRADES`） |
| `createdAt` | ISO string | 必填 | 建檔時間 |
| `courses` | string[] | 可選 | **舊欄位**（2026-06-12 起由 enrollments 取代）。掃描課表仍會寫入，但編輯面板不再寫；保留作 rollback 與歷屆生顯示用 |
| `school` | string | `''` | 學校名稱（消歧用） |
| `birthYear` | number\|null | `null` | 西元出生年 |
| `parentPhone` | string | `''` | 家長電話（消歧用） |
| `status` | string | `'在學'` | 學生狀態：在學/暫停/離開/畢業 |
| `statusChangedAt` | ISO string | 建檔時間 | 上次 status 變更時間 |
| `statusNote` | string | `''` | status 變更備註（為什麼畢業/離開/暫停） |

### Fallback 規則

2026-05-29 前建立的學生**沒有後 6 個新欄位**，讀取時用以下 fallback：

```js
const status = s.status || '在學';
const school = s.school || '';
const parentPhone = s.parentPhone || '';
const birthYear = s.birthYear ?? null;
const statusNote = s.statusNote || '';
const statusChangedAt = s.statusChangedAt || s.createdAt;
```

新欄位**不批次回填**——使用者透過編輯介面慢慢補。已建檔的舊學生在學生卡會出現「📝 待補資料」chip（school 與 parentPhone 都空才會跳）。

### status 列舉值

| 值 | 意義 | 主清單顯示 | 何時用 |
|---|---|---|---|
| `在學` | 目前上課中 | ✓ 在學分頁 | 預設 |
| `暫停` | 暫時不上但會回來 | 歷屆分頁 → 暫停區 | 短期休息 |
| `離開` | 退費/沒說明 | 歷屆分頁 → 離開區 | 中途不繼續 |
| `畢業` | 正向結束 | 歷屆分頁 → 畢業區 | 完整結業 |

狀態間可互轉（含歷屆→在學 = 復學）。`statusChangedAt` 只記最新一次變更時間，**沒有完整 audit log**（如需歷史時間軸，未來階段再加）。

### 同名同年級規則（2026-06-01 起）

`addStudent` 與 `saveStudentEdit` 都會檢查：若要產生同名同年級學生，**所有同名者（含本人）必須至少有一個辨識欄位（school 或 parentPhone）已填**。沒填會被擋下。

### 不存在 studentList 的東西

- **enrollment（課程歸屬）**：2026-06-12 起抽到 `driveData.enrollments`（見下方章節），是「學生 × 課程」的事實來源；`s.courses` 降為舊欄位
- **出席/請假紀錄**：以 Calendar 事件為事實來源，每次渲染從 makeupList 算
- **成績**：尚未實作

## driveData.enrollments（修課登記簿，2026-06-12 起）

每筆 = 一條「誰、修什麼課、哪個期別、什麼價」的紀錄，是學生 × 課程歸屬的**事實來源**（Calendar 備註降為參考資訊）。

| 欄位 | 型別 | 預設 | 說明 |
|---|---|---|---|
| `id` | number | 必填 | 單調遞增計數器（同 studentList 慣例） |
| `studentId` | number | 必填 | 連 `studentList.id`，同名問題在這層終結 |
| `courseTitle` | string | 必填 | 對應 Calendar 事件課名（`origTitle`） |
| `periodId` | string | 必填 | **帶學年**的期別 id，格式 `'2025-sem2'`（學年-期別），跨學年可查歷史 |
| `price` | number\|null | `null` | 每堂單價。`null` = 用價目表預設；數字 = 個人覆蓋（優惠價）。`0` 是合法價格（免費） |
| `startDate` | ISO\|null | `null` | `null` = 期初就上；插班生才填 |
| `endDate` | ISO\|null | `null` | `null` = 上到期末；中途停課才填（學費結算時據此裁切堂數） |
| `note` | string | `''` | 備註 |
| `createdAt` | ISO string | 必填 | 建立時間 |

### 寫入路徑

- **學生編輯面板**：加課/退課/改價，按「儲存」才整批寫回（暫存 `_editEnrollments`，取消即丟棄）
- **掃描課表**：「加入」「更新課程」時順手補登記簿缺的課（`ensureEnrollments`，**只補不刪**——自動刪會弄丟覆蓋價，刪除走編輯面板或之後的對帳工具）
- **一次性轉換**（`migrateCoursesToEnrollments`）：登入載入資料後執行，把在學學生的 `s.courses` 轉成本期 enrollment（price 留 null）。安全閥：有 `enrollmentsMigratedAt` marker 不重跑；studentList 為空（可能雲端讀取失敗）不跑也不寫 marker

### 單價解析順序（`effectivePrice`）

個人覆蓋（`en.price`）→ 價目表預設（`coursePrices`）→ 都沒有 = `null`（未定價，UI 紅字提示）

## driveData.coursePrices（課程價目表，2026-06-12 起）

每門課的預設每堂單價，學生管理頁「💰 價目表」編輯：

| 欄位 | 型別 | 說明 |
|---|---|---|
| `title` | string | 課名（對應 Calendar 事件課名） |
| `price` | number\|null | 預設每堂單價；`null` = 課名先建、價格未定 |

## driveData.makeupScheduled（補課排程）

每筆對應一個原請假/調課事件 + 已排定的補課時段：

| 欄位 | 型別 | 說明 |
|---|---|---|
| `originalId` | string | 原 Calendar 請假/調課事件 ID |
| `origTitle` | string | 原課程名稱 |
| `originalDate` | ISO string | 原請假/調課時間 |
| `scheduledDate` | ISO string | 補課時段起點 |
| `scheduledEnd` | ISO string | 補課時段終點 |
| `room` | string | 補課教室 |
| `calEventId` | string\|null | 新建立的補課 Calendar 事件 ID |
| `absentStudents` | string[] | 請假學生**名字**（字串陣列，**沒有 id 連結**） |
| `calName` | string | 寫入哪本行事曆（補課/調課） |

### 為什麼 `absentStudents` 還是字串陣列？

Calendar 備註本身是字串，沒有 student id 概念。要 id 化的話，得在解析時做名字 → id 的對應——而同名問題會在這層爆。enrollment 資料層 2026-06-12 已建立，等「課程卡改讀 enrollment」「掃描轉對帳」完成後這裡才能徹底 id 化。

目前的同名 workaround：在 Calendar 備註寫「（國小）子晴」，parseScanName 會拆出 name + gradeHint。但仍限於配對掃描，stats 計算（`getStudentStats`）只用名字字串比對。

## driveData 結構

```js
driveData = {
  studentList: [...],          // 上面定義
  makeupScheduled: [...],      // 上面定義
  enrollments: [...],          // 修課登記簿（2026-06-12 起）
  coursePrices: [...],         // 課程價目表（2026-06-12 起）
  enrollmentsMigratedAt: ISO,  // 一次性轉換 marker
}
```

存在 Firestore，登入後讀取、寫入時 debounce ~1.5 秒自動同步（`scheduleDriveSave`）。
