## driveData.studentList（學生主清單）

每位學生一個物件，欄位如下（2026-06-01 起）：

| 欄位 | 型別 | 預設 | 說明 |
|---|---|---|---|
| `id` | number | 必填 | `Date.now()` 產生，唯一識別 |
| `name` | string | 必填 | 學生姓名 |
| `grade` | string | 必填 | 國小/國一/國二/國三/高一/高二/高三/大學（見 `GRADES`） |
| `createdAt` | ISO string | 必填 | 建檔時間 |
| `courses` | string[] | 可選 | **舊欄位（已凍結）**。2026-06-12 起由 enrollments 取代，編輯面板與課表對帳皆不再寫入；保留作 rollback 與歷屆生顯示用 |
| `school` | string | `''` | 學校名稱（消歧用） |
| `birthYear` | number\|null | `null` | 西元出生年 |
| `parentPhone` | string | `''` | 家長電話（消歧用） |
| `status` | string | `'在學'` | 學生狀態：在學/暫停/離開/畢業 |
| `statusChangedAt` | ISO string | 建檔時間 | 上次 status 變更時間 |
| `statusNote` | string | `''` | status 變更備註（為什麼畢業/離開/暫停） |

### Fallback 規則

2026-05-29 前建立的學生**沒有後 6 個新欄位**，讀取時用以下 fallback：

```js
const status = s.status || '在學';
const school = s.school || '';
const parentPhone = s.parentPhone || '';
const birthYear = s.birthYear ?? null;
const statusNote = s.statusNote || '';
const statusChangedAt = s.statusChangedAt || s.createdAt;
```

新欄位**不批次回填**——使用者透過編輯介面慢慢補。已建檔的舊學生在學生卡會出現「📝 待補資料」chip（school 與 parentPhone 都空才會跳）。

### status 列舉值

| 值 | 意義 | 主清單顯示 | 何時用 |
|---|---|---|---|
| `在學` | 目前上課中 | ✓ 在學分頁 | 預設 |
| `暫停` | 暫時不上但會回來 | 歷屆分頁 → 暫停區 | 短期休息 |
| `離開` | 退費/沒說明 | 歷屆分頁 → 離開區 | 中途不繼續 |
| `畢業` | 正向結束 | 歷屆分頁 → 畢業區 | 完整結業 |

狀態間可互轉（含歷屆→在學 = 復學）。`statusChangedAt` 只記最新一次變更時間，**沒有完整 audit log**（如需歷史時間軸，未來階段再加）。

### 同名同年級規則（2026-06-01 起）

`addStudent` 與 `saveStudentEdit` 都會檢查：若要產生同名同年級學生，**所有同名者（含本人）必須至少有一個辨識欄位（school 或 parentPhone）已填**。沒填會被擋下。

> ⚠️ 這個檢查是**逐字比對 `name`**。若把年級括號塞進名字欄（如「（國小）子晴」vs「子晴」），兩者字串不同 → 不被視為同名 → 警告失效、重複建檔擋不下來。括號年級的正確位置是 **Calendar 備註**（掃描時 `parseScanName` 會拆成 name + gradeHint），**不是名字欄**。舊版掃描曾把括號寫進名字，造成同一人重複，需用硬刪除清理。

### 硬刪除（2026-06-13 起）

學生編輯面板「危險區」可**永久刪除**學生（`confirmDeleteStudent`，需打對姓名）：

- 從 `studentList` 移除該學生
- 連動 `saveEnrollments` 移除**該 studentId 的所有 enrollments**（不分期別）
- `makeupScheduled` 以名字字串記缺席者、**無 studentId 連結**，不在此清理（避免同名誤刪；屬 Calendar 衍生歷史）
- 不碰 Google Calendar
- 與軟刪除（`status` 改歷屆）並存：正常畢業/離開走軟刪除，硬刪除只給清測試帳號／重複資料用

### 不存在 studentList 的東西

- **enrollment（課程歸屬）**：2026-06-12 起抽到 `driveData.enrollments`（見下方章節），是「學生 × 課程」的事實來源；`s.courses` 降為舊欄位
- **出席/請假紀錄**：請假/曠課以 Calendar 事件為事實來源；出席（點名）2026-06-19 起獨立成「按期別分文件」（見下方章節），**不存 studentList、不存 driveData**
- **成績**：2026-06-19 起獨立成「按期別分文件」（見下方章節）

## driveData.enrollments（修課登記簿，2026-06-12 起）

每筆 = 一條「誰、修什麼課、哪個期別、什麼價」的紀錄，是學生 × 課程歸屬的**事實來源**（Calendar 備註降為參考資訊）。

| 欄位 | 型別 | 預設 | 說明 |
|---|---|---|---|
| `id` | number | 必填 | 單調遞增計數器（同 studentList 慣例） |
| `studentId` | number | 必填 | 連 `studentList.id`，同名問題在這層終結 |
| `courseTitle` | string | 必填 | 對應 Calendar 事件課名（`origTitle`） |
| `periodId` | string | 必填 | **帶學年**的期別 id，格式 `'2025-sem2'`（學年-期別），跨學年可查歷史 |
| `price` | number\|null | `null` | 每堂單價。`null` = 用價目表預設；數字 = 個人覆蓋（優惠價）。`0` 是合法價格（免費） |
| `startDate` | ISO\|null | `null` | `null` = 期初就上；插班生才填 |
| `endDate` | ISO\|null | `null` | `null` = 上到期末；中途停課才填（學費結算時據此裁切堂數） |
| `note` | string | `''` | 備註 |
| `createdAt` | ISO string | 必填 | 建立時間 |

### 寫入路徑

- **學生編輯面板**：加課/退課/改價，按「儲存」才整批寫回（暫存 `_editEnrollments`，取消即丟棄）
- **課表對帳**：「建檔並登記」「補登」寫入（`ensureEnrollments`）；「退課」移除該筆 enrollment（對帳是目前唯一的批次退課入口）；每一條都由使用者逐項確認才動資料
- **一次性轉換**（`migrateCoursesToEnrollments`）：登入載入資料後執行，把在學學生的 `s.courses` 轉成本期 enrollment（price 留 null）。安全閥：有 `enrollmentsMigratedAt` marker 不重跑；studentList 為空（可能雲端讀取失敗）不跑也不寫 marker

### 單價解析順序（`effectivePrice`）

個人覆蓋（`en.price`）→ 價目表預設（`coursePrices`）→ 都沒有 = `null`（未定價，UI 紅字提示）

## driveData.makeupScheduled（補課排程）

每筆對應一個原請假/調課事件 + 已排定的補課時段：

| 欄位 | 型別 | 說明 |
|---|---|---|
| `originalId` | string | 原 Calendar 請假/調課事件 ID |
| `origTitle` | string | 原課程名稱 |
| `originalDate` | ISO string | 原請假/調課時間 |
| `scheduledDate` | ISO string | 補課時段起點 |
| `scheduledEnd` | ISO string | 補課時段終點 |
| `room` | string | 補課教室 |
| `calEventId` | string\|null | 補課 Calendar 事件 ID（**2026-07-17 第 3 刀起新紀錄一律 `null`**——不再建 Calendar 事件，主頁場次由展開器從本紀錄直接長出；舊紀錄取消安排時仍會刪對應事件） |
| `absentStudents` | string[] | 請假學生**名字**（字串陣列，**沒有 id 連結**） |
| `calName` | string | 寫入哪本行事曆（補課/調課） |

### 為什麼 `absentStudents` 還是字串陣列？

Calendar 備註本身是字串，沒有 student id 概念。要 id 化的話，得在解析時做名字 → id 的對應——而同名問題會在這層爆。enrollment 資料層 2026-06-12 已建立，等「課程卡改讀 enrollment」「掃描轉對帳」完成後這裡才能徹底 id 化。

目前的同名 workaround：在 Calendar 備註寫「（國小）子晴」，parseScanName 會拆出 name + gradeHint。但仍限於配對掃描，stats 計算（`getStudentStats`）只用名字字串比對。

## 系統自有課表（2026-07-04 老闆整包拍板，尚未實作）

> **這是什麼**：開發路線 ①「系統擁有課表」的 schema。設計原則＝**旋鈕不是箱子**：不先選類別，填自由欄位（誰/科目/老師/排法/教室/費用），系統自動判類型、自動命名（可手改）。**全部不寫回 Google Calendar**，系統自己是唯一真相。
> **生效時機**：cutover 時啟用（先建功能 → 全頁改讀系統 → 清空重建；絕不可先清）。

### driveData.courses（課程，每門一筆）

| 欄位 | 型別 | 預設 | 說明 |
|---|---|---|---|
| `id` | number | 必填 | 穩定識別（`Date.now()` 慣例）。**課程身分＝id，不再拿課名當 join key** |
| `name` | string | 自動命名 | 依內容自動長：1人「(學生)(科目)家教」、2人「(學生)、(學生)(科目)班」、3+「(年級)(科目)班」、試聽「(學生)(科目)試聽」；**可手改** |
| `type` | string | 自動 | 自動長出的標籤：`一對一`/`一對二`/`團班`/`練習課`/`試聽`（不是限制，任何組合合法） |
| `teacherIds` | number[] | 必填 | 連 `teachers.id` 的**陣列（可多位老師）**；顯示以「、」串。舊資料的單一 `teacherId` 由 `courseTeacherIds()` 自動讀成 `[teacherId]`（向後相容）。練習課＝預設輔導老師 |
| `teacherRate` | number\|null | `null` | **老師費率**（薪資表用）：團班＝每人每堂、家教＝每小時、補課/試聽＝每堂固定。`null`＝未定 |
| `schedule` | object | 必填 | `{mode:'weekly'\|'dates', slots:[...], phases:[...]}`；weekly slot＝`{weekday, start, end}`、dates slot＝`{date, start, end}`；**可多 slot**。`slots`＝起始段（課程建立起生效）。`phases`＝**換時段**（多段上課時間，僅 weekly）：`[{from:'YYYY-MM-DD', slots:[{weekday,start,end}]}]`，同一門課不同期別時間不同時用，展開器（schedule.js `_schedulePhases`/`_activePhase`）依日期自動套用「from≤該天的最後一段」；無 phases＝單段、行為同舊。切割點通常＝學期/假期交界（表單有暑假7/1・上學期9/1・寒假2/1・下學期3/1 快捷） |
| `room` | string | `''` | 教室；選項＝北投 5 間（大教室/小教室/108/208/309）＋`石牌分校`（`COURSE_ROOMS`）。`石牌分校`＝補課/調課分校判斷的鑰匙（`ev.classroom==='石牌分校'`→石牌），不進主頁北投時間軸 |
| `defaultPrice` | number\|null | `null` | 每堂收費預設（個人覆蓋在 enrollment）；**練習課/試聽＝`null` 且不結學費** |
| `needsGrade` | boolean | `false` | 需登記成績（**練習課預設 `true`**） |
| `subject` | string | `''` | 科目標記（高中作業清單、課名沒含科目時用） |
| `sourceChannel` | string | `''` | **試聽課專屬**：來源管道（下拉常用選項＋可自填） |
| `status` | string | `'開課中'` | 開課中/已結束 |
| `createdAt` | ISO string | 必填 | 建立時間 |

### driveData.absences（系統請假紀錄，2026-07-17 起・全頁改讀系統第 2 刀）

系統課堂的請假/曠課不再寫 Google Calendar 標題，改存這裡。**每課堂一筆**（以課堂合成 id 為鍵）：

| 欄位 | 型別 | 說明 |
|---|---|---|
| `id` | number | 識別（Date.now() 慣例） |
| `occId` | string | 課堂合成 id：`sys:<courseId>:<YYYY-MM-DD>:<slot 序>`（重載不變；點名紀錄同此鍵） |
| `courseId` | number | 連 `courses.id`（刪課連動刪紀錄） |
| `date` | ISO string | 課堂開始時間（期別篩選、待補課清單重建課堂用） |
| `teacherAbsent` | boolean | 老師請假（true 時 `leave`/`noShow` 清空，同標題格式時代慣例） |
| `leave` | array | 請假學生 `[{studentId, name, timing:'A'\|'B'}]` |
| `noShow` | array | 曠課學生 `[{studentId, name}]`（時機 C＝曠課，展開時 timing 補 'C'） |
| `makeupSkip` | string[] | 「不補課・退半堂」學生**名字**（取消請假時同步剔除不再請假者） |
| `resched` | boolean | **調課旗標（2026-07-17 第 3 刀起）**：true＝整堂移走（展開時同 parseEv 語意：absType=調課、absentStudents=全名冊、isFullAbsent） |
| `reschedReason` | string | 調課原因（選填） |
| `createdAt` / `updatedAt` | ISO string | 建立／更新時間 |

規則：
- **名單雙存 studentId＋name**：id 供學費/薪資結算（同名終結）、name 供顯示與既有 UI（請假面板、統計皆名字制）
- **紀錄清空即刪**：teacherAbsent=false 且 leave/noShow 皆空**且 resched=false** → 整筆刪除（無殘留空紀錄）
- 展開器（schedule.js）把紀錄疊回課堂物件（isAbsent/absentStudents/absenceTiming/... 語意對齊 parseEv），下游 UI 不分行事曆課/系統課
- 待補課清單：`sysAbsenceEvents()` 逐筆重建該課堂進 makeupList；課程已刪的紀錄跳過
- 存放於 `sharedData/main` 主文件（同 driveData 其他欄），firestore.rules 不需更動
- ⚠️ cutover 清空工具的整份重寫白名單**不含此欄**（工具已完成使命待刪，勿再執行）

|---|---|---|
| `studentId` | number | 哪位學生 |
| `courseId` | number | 哪門課（學費單一列一課） |
| `sessions` | number | 上課堂數（自動＝該月 attendance 數，可手改） |
| `unitPrice` | number | 單堂費用（自動＝effectivePrice，可手改） |
| `adjust` | number | 溢繳/補繳費用（**手動**，可負） |
| `note` | string | 備註（如「教材費」） |

本月學費＝`sessions × unitPrice + adjust`（顯示時算，不存）。學費單輸出：照老闆模板、每格可手改、**一鍵輸出圖片**。收據功能之後另做（格式待老闆給）。

### 待補課清單（makeupScheduled）

照舊保留——**cutover 唯一倖存者**。以名字字串記，重建後靠名字重對回新 studentId（見上方該章節）。

## 點名 / 成績 / 作業（按期別分文件，2026-06-19 起，尚未實作）

> **為什麼不放 driveData**：50 生 × 每週 5 堂 × 40 週 ≈ 1 萬筆點名/年，連同成績、作業塞單一 `driveData` 文件會在一學年內撐爆 Firestore **1MB 單文件上限**。改成**每資料類型 × 每期別各一份文件**（`attendance_2025-sem2` 等），用到才載入：單份有界（每學期約 400KB）、契合現有「整包載入」模型、且 period-scoped 天然支援「學年封存」🔴 需求。
> **periodId 格式**：同 enrollment 的帶學年期別 id（`'2025-sem2'`）。

### attendance_&lt;periodId&gt;（點名）

| 欄位 | 型別 | 說明 |
|---|---|---|
| `eventId` | string | Calendar 事件 instance ID（哪一堂） |
| `date` | ISO string | 上課日（查詢/封存用，免去解析 eventId） |
| `studentId` | number | 連 `studentList.id`，同名問題在此終結 |
| `status` | `'到'` | 有來＝出席（含遲到）。一筆紀錄＝出席一堂 |
| `lateMin` | number | 遲到分鐘數；`0`＝準時、`>0`＝遲到 N 分（**仍算出席、仍算一堂**） |
| `markedAt` | ISO string | 點名時間 |

- **狀態模型（2026-06-19 改）**：實務上「沒按到」的人多半是**遲到**（要記分鐘），真正整堂沒來很罕見、且那就是**曠課**。所以 attendance 只記出席（`status` 一律 `'到'`，`lateMin` 區分準時/遲到），**不再有 `'未到'`**——沒來一律走既有曠課流程（標記請假 → 時機「已開始·曠課」→ 寫回 Calendar 標題 `【X曠課】`）。曠課以 Calendar 為事實來源，不在 attendance 重複記，避免兩套打架。
- **學費堂數來源**：該生該期 attendance 紀錄數（都算出席，含遲到）。取代「排定數 − 不補課的請假/曠課」推算。
- **試聽課不點名**：試聽不算堂數、不結學費，不進點名面板。
- **UI**：今日課程卡內嵌點名面板——「全部到」一鍵、每生「到／遲到（選分鐘 5/10/15/20/30 或自訂）」、罕見沒來按「曠」跳曠課流程；請假/曠課的學生自動帶入鎖定。

### grades_&lt;periodId&gt;（成績，每堂每生可多筆）

| 欄位 | 型別 | 說明 |
|---|---|---|
| `eventId` | string | 哪一堂 |
| `studentId` | number | 連 `studentList.id` |
| `label` | string | 標籤（如「課前考」「第二份考卷」） |
| `score` | number\|null | 分數；`null` = 只記標籤未打分 |
| `note` | string | 備註 |
| `createdAt` | ISO string | 建立時間 |

- 多筆設計：涵蓋課前考、練習課重寫第二份考卷等同一堂多次登記的情境。
- 哪些課要登成績：靠課程設定的「需登記成績」開關（新課預設只點名，見 [`todos.md`](todos.md) 開發路線）。
- **已實作（2026-07-17）**：增欄 `id`（number，刪除用）；`eventId`＝系統課堂合成 id（`sys:courseId:日期:slot 序`）；儲存模式同 attendance（用到才載入、debounce 1.5 秒寫回、`{merge:true}`）。

### exams_&lt;periodId&gt;（段考成績，2026-07-17 起）

學校段考，與課堂無關、以學生為單位手動登記（學生視窗「成績」區）。

| 欄位 | 型別 | 說明 |
|---|---|---|
| `id` | number | 識別（刪除用） |
| `studentId` | number | 連 `studentList.id` |
| `examName` | string | 段考名稱（自由填，如「一段」「期末」） |
| `subject` | string | 科目（可空） |
| `score` | number\|null | 分數；`null`＝只記名稱未打分 |
| `note` | string | 備註（保留欄，UI 尚未開） |
| `createdAt` | ISO string | 建立時間 |

儲存模式同 grades/attendance：`sharedData/exams_<yearPeriodId>` 的 `{records:[…]}`，用到才載入（學生視窗開啟時）、debounce 1.5 秒寫回。

### homework_&lt;periodId&gt;（高中數學作業，每天一張清單）

| 欄位 | 型別 | 說明 |
|---|---|---|
| `date` | ISO string | 哪一天 |
| `studentId` | number | 連 `studentList.id` |
| `courseTitle` | string | 哪門數學課 |
| `status` | `'已交'`\|`'未交'` | 作業狀態 |
| `note` | string | 備註 |

- 清單來源：在學高中生 × 本期有修數學課（課名沒含「數學」的數學課靠課程設定補標科目）。
